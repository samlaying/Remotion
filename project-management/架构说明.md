# Remotion 项目管理系统架构说明

## 🏗️ 系统架构

这是一个分层架构的视频项目管理系统，设计目标是支持多项目、可扩展、易维护。

---

## 📊 架构层次

```
┌─────────────────────────────────────────┐
│           用户界面层 (UI)                │
│     Remotion Studio / Browser           │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         项目管理层 (Management)          │
│         Root.tsx / Project Manager      │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         项目执行层 (Projects)            │
│    RemotionGuide / Custom Projects      │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│          核心引擎层 (Core)               │
│        Remotion / React / Zod           │
└─────────────────────────────────────────┘
```

---

## 🔄 数据流架构

### 1. 项目初始化流程

```
projects.json → Project Manager → Root.tsx → Studio
     ↓              ↓                ↓
  配置数据      数据验证        组件注册
```

### 2. 视频渲染流程

```
Studio 选择项目 → 加载组件 → 验证 Props → 渲染帧 → 输出视频
     ↓               ↓          ↓          ↓         ↓
  项目列表      组件实现    Schema验证   动画计算   MP4输出
```

### 3. 项目管理流程

```
用户操作 → API 调用 → 数据更新 → 状态同步 → 界面刷新
   ↓          ↓          ↓          ↓          ↓
创建/编辑   Manager函数  JSON更新   重新加载    Studio
```

## 📁 核心模块

### 1. 类型系统 (types/project.ts)

**职责**：定义所有数据结构和类型

```typescript
// 核心类型
├── Project              # 项目实体类型
├── Template             # 模板类型
├── ProjectsData         # 项目集合类型
└── Schema验证           # Zod Schema 定义
```

**设计原则**：
- 类型安全优先
- 支持 Zod 验证
- 清晰的接口定义

### 2. 项目管理器 (lib/projectManager.ts)

**职责**：提供项目管理核心功能

```typescript
// 核心功能
├── loadProjectsData()        # 加载项目数据
├── getAllProjects()          # 获取所有项目
├── getActiveProjects()       # 获取活跃项目
├── getProjectById()          # 根据 ID 获取项目
├── createProject()           # 创建新项目
├── updateProject()           # 更新项目
├── deleteProject()           # 删除项目
└── createProjectFromTemplate() # 从模板创建
```

**设计模式**：
- 单一职责原则
- 函数式编程
- 不可变数据

### 3. 项目根组件 (Root.tsx)

**职责**：管理所有项目的 Composition 注册

```typescript
// 核心功能
├── 加载活跃项目列表
├── 动态注册 Composition
├── 管理项目配置
└── 处理项目切换
```

**扩展方式**：
```typescript
// 添加新项目
import { NewProject, newProjectSchema } from "./projects/NewProject";

<Composition
  id="NewProject"
  component={NewProject}
  schema={newProjectSchema}
  // ... 配置
/>
```

### 4. 项目实现层 (projects/)

**职责**：实现具体的视频项目

```
projects/
├── RemotionGuide/          # Remotion 技能指南
│   ├── index.tsx          # 主组件
│   ├── config.ts          # 项目配置
│   └── schema.ts          # 数据验证
├── CustomProject/         # 自定义项目
│   ├── index.tsx
│   ├── config.ts
│   └── scenes/            # 场景组件
│       ├── Scene1.tsx
│       └── Scene2.tsx
└── [更多项目...]
```

## 🔐 数据安全与验证

### 1. Schema 验证流程

```typescript
用户输入 → Zod Schema → 类型推断 → 运行时验证 → 安全使用
   ↓          ↓           ↓            ↓           ↓
 defaultProps  Schema    Props类型    验证通过    组件渲染
```

### 2. 类型安全保障

```typescript
// 1. 定义 Schema
export const projectSchema = z.object({
  title: z.string(),
  features: z.array(z.string()).length(3),
});

// 2. 推断类型
type ProjectProps = z.infer<typeof projectSchema>;

// 3. 使用类型
const MyComponent = ({ title, features }: ProjectProps) => {
  // 完全类型安全
  features.map(f => f.toUpperCase()) // TypeScript 知道这是 string[]
};
```

### 3. 配置验证

```typescript
// 项目配置验证
const configSchema = z.object({
  durationInFrames: z.number().positive(),
  fps: z.number().min(1).max(120),
  width: z.number().positive(),
  height: z.number().positive(),
});
```
